CSCI3180 - Subprogram

Created2023-04-27|Updated2023-04-27|CSCI3180
|Post View:

Introduction

Procedure & Function

  • Subprograms are collections of statements that define parameterized computations
  • There are two kinds of subprograms:
    1. Functions return values
    2. Procedure do not
  • In most of the languages do not include procedures as a separate form of subprogram (functions that can be defined not to return values can be used as procedures)

Execution

  • Each subprogram has a single entry point
  • The calling subprogram is suspended during execution of the called subprogram (i.e., there is only one subprogram in execution in the given time)
  • Except the program is terminated, control always return to the caller at the subprogram called statement when the called subprogram’s execution terminate

Definition

  • Subprogram Definition

    • describes the interface to and the actions of the subprogram abstraction

      • f is the definition of this subprogram

    • In Python, function definitions are executable

    • In all other languages, they are non-executable

    • Executable function definition:

  • Subprogram Call

    • an explicit request that the subprogram be executed (fun())

  • Subprogram Header

    • the first part of the definition, including the name, the kind of subprogram, and the parameters

  • Parameter Profile

    • the number, order, and types of its parameters

  • Protocol

    • the parameter profile and the return type

  • Subprogram Declaration

    • the protocol of the subprogram (prototype in C/C++)

  • Parameter (formal parameter)

    • a variable listed in the subprogram header and used in the subprogram

  • Argument (actual parameter)

    • the actual input supplied in the subprogram call statement to the subprogram

Correspondence

  1. Positional

    • the binding of arguments to parameters is by position
    • Advantage: Simple and effective

  2. Keyword (in Python, JavaScript, C#)

    • the name of the parameter is in form of a keyword, to which an argument is to be bound is specified with the argument
    • Advantage: arguments can appear in different order
    • Disadvantage: users are able to know the parameter’s names

Parameter Default Values

  • parameters can have default values if no argument is passed
  • Note: the default parameters must appear last(because parameters are positionally associated)
  • Remind: the default parameters should be assign in subprogram declaration (if have)

    • default parameter in C++

    • list of variable number of parameters in Python (*)

    • named keyword argument in Python (**)

Parameter Passing Methods

Passing Value

  1. Pass-by-Value

    • the value of the argument is used to initialize the corresponding parameter
    • normally implemented by simply copying
    • Advantage: no side-effect
    • Disadvantage: require additional storage

  2. Pass-by-Result

    • no value is transmitted to the subprogram, the parameter simply acts as a local variable
    • i.e., the value is transmitted to the caller’s argument when control return to the caller by coping
    • Potential problem:
      call foo(a, a): in this case, what should a be?

  3. Pass-by-Value-Result

    • combination of pass-by-value and pass-by-result
    • i.e., copying the value of argument to the parameter, then do some change on the parameter in the subprogram, then copying the value of parameter to the argument when return
      • Disadvantage: combination of pass-by-value and pass-by-result

Passing Object

  1. Pass-by-Reference

    • pass the reference to the argument variable (can be simulated in pass-by-value of pointers for languages that not support reference)
      • copying the address of arguments instead of the whole data structure
    • Advantage: passing is efficient because there is no copying and duplicate storage
    • Disadvantage:
      • slower access (indirect access)
      • potential for the side effect

  2. Pass-by-Sharing

    • copy the reference argument (point to an object) and pass to the parameter
    • Note: it is for object only (so need to wrapper the primitive data values in we want to use Pass-by-Sharing in those values)
    • Python / Ruby use Pass-by-Sharing
      Everything is concidered as an object (all variables are considered as references to some objects)
    • Java also use pass-by-sharing for object arguments
      Consider Pass-by-Sharing is just similar to Pass-by-value (copying), so simply use Pass-by-Value (where the value is a reference)
  • Pass-by-Renerence vs Pass-by-Sharing

    • Implementation of Pass-by-Renerence (similated using pointer) in C++

    • Implementation of Pass-by-Sharing in Java

    • The main different is the new assignment for the parameter in the subprogram:

      • Pass-by-Reference is passing a reference to the argument
      • Pass-by-Sharing is passing a copied reference from the argument
        Therefore, when there is a new assignment to the parameter, the argument is also changed in Pass-by-Reference but not Pass-by-Sharing

Parameters

Multidimensional Arrays

  • access an element in a multi-dimensional array, a storage mapping function (address calculation) has to be built

    • int a[4][2] = {{11,12}, {21,22}, {31,32}, {41,42}};
      Memory Address (One-dimensional)
      11 12 21 22 31 32 41 42 ...
      • address of a[3][1] = (address of a[0][0]) + (3 * 2 + 1) * (sizeof int)

  • C/C++ Implementation

    • Porblems:

      1. C/C++ requires programmer to include the declared size of all (but not the first dimension) in the array parameter

        • It is necessary for the programmer to declare the parameter like
          void f(int a[2][3][8]) or void f(int a[][3][8])(but not the first dimension)

      2. sizeof array in a function

    • Solution:
      • We can use a flexible workaround
        • pass a pointer to the array and the sizes of the dimensions as other parameters
          void f(int* a, int firstDSize, int secondDSize, int thirdDSize)

  • Java/C#/Python Implementation

    • Arrays are objects in those languages, so the multi-dimensional array is single-dimensioned
      • int a[][] = {{11}, {21,22,23}, {31,32}};
        Memory Address (One-dimensional)
        Array Object Array Object Array Object ...
        11 21 22 23 31 32 ...
      • Access the size of array:
        • l.length in Java
        • len(l) in Python

Subprograms

  • There are situations in which a subprogram is a parameter of another subprogram

  • C/C++ Implementation

    • functions can be passed as function pointers (*p)(...)

  • Python Implementation

    • functions in Python are first-class citizens and simply objects
      so it can be simply passed as parameter like any other Objects

Closures

  • it is a subprogram and the referencing environment where it was defined
  • the referencing environment is needed if the subprogram can be called from any arbitrary place in the program

  1. A static-scoped language that does not permit nested subprograms doesn’t need closures

    • Implementation in C++:
      • the referencing environment in f (called from any arbitrary place in the program) is the local variables (a, b, c) and the global variables which is same as directly calling, so the static-scoped language that does not support nested programming doesn’t need closures

  2. Closures are necessary for a subprogram that can access variables in the nesting scopes and it can called from abywhere

    • Implementation in Python:
      • the referencing environment of template_adder has x which is outside the definition of template_adder, but because Python supports closures, so x is also passed through to the template_adder
      • the clousre returned from adder is both the function and the referencing environment
  • closures actually extended lifetime to some variables, in case, x is extended as the function adder has already ended

Lambdas

  • anonymous functions (which syntax is much more simplier and readable than defining a function) using the keyword lambda
    • Implementation in Python:
      • In case, the function named template_adder is not important as it would be “rename” later
        (add_five = adder(5): the function template_adder is actually renamed to add_five)

Polymorphic Subprograms

  • Polymorphic subprograms are subprograms that can operate on different types of data

  1. Adhoc Polymorphism

    • Overloaded subprograms provide a particular kind of polymorphism called adhoc polymorphism

  2. Subtype Polymorphism

  • A parameter of type X can take any object of type X or any type derived from X
