CSCI3180 - Logic Programming

Created2023-04-28|Updated2023-04-28|CSCI3180
|Post View:

Declarative Programming

  1. Imperative (Conventional) Languages Properties

    • State-oriented: each statement execution changes the abstract machine state
    • Destructive assignment: update the value
    • Side effects
    • Difficult to read, write, and verify programs

  2. Declarative languages

    • Higher program understandability and verifiability
    • Referential transparency: no side effect

Prolog

  • Divided into two stages:

    1. Building a program (asserting what is true)
    2. Running a program (asking for consequences of what has been asserted)

  • A Prolog program is a collection of assertions

    1. Facts
    2. Rules
      They are used to express relationships amongst some objects

Facts

  • a statement that asserts a relationship between objects

  • Facts can be of various arities

  • Unary facts denote properties

Query

  • Prolog is executed by posting a question (query), which makes a request for information based on the facts and rules defined in a program.

    • provided facts
    • query

  • In general, query can carry many subqueries:
    $$\text{?-} \space G_1, \cdots , G_m \quad \forall m \geq 0 \quad \text{where G is an query}$$

    • use , to conjunct the subqueries
    • When $m=0$, it is an empty query

Terms

  • Prolog programs are constructed from terms which can be constants, variables or structures

  1. Constants

    • represent a specific object
    • must start with a lower-case letter karl
    • they can be number (but we just focus on the logical part so we seldom use number)

  2. Variables

    • must start with a upper-case letter X, Y, Z

  3. Structures

    • They consist of a functor and a number of arguments
      • bonks(big_doge, small_doge)
      • functor: bonks
      • arguments: big_doge, small_doge
    • Advanced usage:
      • Suppose we want to represent a location on a map, we can use a structure with two components a latitude (p) and a longitude (q) loc(p, q)

  4. Functor

    • glue some objects into a composite object
    • A functor may have only one argument (for labeling an argument)
      • south(32)
      • loc(north(45), east(72))

Rules

  • Some concepts are based on the others

    • e.g., To find a parent, we have to ask two questions to obtain a single piece of information

      • either of the above queries succeeds, a parent of Kari is found

    • We can translate the “parent” concept into the “father” and “mother” concepts using Rules

  • Form of Rules:
    $$H \space \text{:-} \space B_1, \cdots, B_n \quad \forall n \geq 0 \quad \text{where H is the head and B is the body of the rule}$$

    • Variables in a rule are universally quantified

    • More exmaples:

    • Rule for ancestor

      • A parent is an ancestor
        ancestor(X, Y) :- parent(X, Y)
      • An ancestor of X: a parent of an ancestor of an individual X
        ancestor(X, Z) :- parent(X, Y), ancestor(Y, Z)

  • Unconditional Rules

    • the rules with no body $n=0$ is an unconditional rule

Answer and Response

  1. If the query has variables

    • the answer subsitution may assign a constant to some of the variables in the query

  2. If the query has no variables

    • the answer subsitution is vacuous (i.e., the query itself is the Answer)

  3. When a program consists of facts only

    • for the correct answer, the answer must literally appear as a fact in the program

  • Reduction

    • A query can be replaced to another by matching the head of the rule.
    • If a sub-query matched a fact, then that sub-query can be reduced and applying the matching substitution to the entire resulting query.
