Statements (Expressions and Assignment)

Created2023-03-13|Updated2023-03-13|CSCI3180
|Post View:

Arithmetic Expressions

  • consist of operators, operands and parentheses

Operators

  • can be infix, prefix, or postfix (most of them are infix)

    1. Unary Operator: has one operator

      • i++, --i

    2. Binary Operator: has two operators

      • a + b

    3. Ternary Operator: has three operators

      • a ? b : c

Operator Precedence Rules

  • determines the order in which operators are evaluated in an expression
  • Typical precedence levels:
    1. Parentheses ()
    2. Exponentiation ** or ^
    3. Multiplication *, /
    4. Addition & Subtraction + & -

Operator Associativity Rule

  • refers to the order in which operators with the same precedence are evaluated in an expression
  • Typical associativity rules:
    • right to left (except exponentiation **)
  • Remind: APL is different: all operators have equal precedence and all operators associate right to left

Operand

  1. Variable
  2. Constant
  3. Parenthesized expression
  4. Function call

Side Effecr

  • it occurs when a function changes a two-way parameter, a non-local variable, or a static-storage variable

    1. updated a non-local variable

    2. updated a static-storage variable

  • Solutions

    1. disallow functional side effects
      • No two-way parameters in functions
      • No static-storage / non-local assignable variable access in function
      • Disadvantages: Inflexibility

Referential Transparency

  • if it always produces the same result for a given set of input parameters, and its result does not depend on any external state or side effects.

    • the function is referential transparency if result1 and result2 are equivalent

  • Advantages

    • much easier to understand as they do not have variables

Overloaded Operators

  • an operator can work for more than one data type or class

    • 1 + 2: work for int
    • 1.1 + 2.2: also work for float
    • &a: getting the address of a
    • a&b: logic and operation

  • C++, C#, and F# allow user-defined overloaded operators

    • When sensibly used, such operators can be an aid to readability

    • Potential problems:

      1. users can define nonsense operations

      2. readability may suffer, even when the operators make sense

Type Conversions

  1. narrowing conversion (Not allow in Java)

    • the converted object cannot include all of the values of the original type by the programming language
    • float to int

  2. widening conversion

    • the converted object can include at least approximations to all of the values of the original type
    • int to float

Implicit Type Conversions

  • automatic conversion of a value from one data type to another
  • Advantage: convenience
  • Disadvantage: reliability (lower the error detection ability of the compiler)

Explicit Type Conversions

  • manual conversion of a value from one data type to another by the programmer
  • int(variable) or (int)variable: depend on language

Relational and Boolean Expressions

Relational Expressions

  • the comparation of two operands with one relational operator and evaluate to Boolean
  • some operator symbols which vary among languages (!=, ne, <>)
  • JavaScript and PHP have two additional relational operator === and !==

Boolean Expressions

  • The operands and the result are all Boolean

    • C, C++, Java: &&, ||
    • Python: and, or

  • Traditional C: Boolean true value is 1 whereas Boolean false value is 0

  • Chain comparison:

Short-Circuit Evaluation

  • an expression in which the result is determined without evaluating all of the operands and/or operators

    • (87 * a) * (b / 17 - 9): if a is 0, then no need to evaluate (b / 17 - 9)
    • A && B: if A is false, then no need to evaluate B
    • A || B: if A is true, then no need to evaluate B

  • bitwise Boolean operators that are not short-circuit: & and |

  • Advantages: efficiency (save the compilation power)

  • Condition: program correctness

    • if we use the & operator that doest not has short-circult
    • it is compilation error, because when index is 4, a[index] does not exist

  • Potential error: (a > b) || (b++ / 5) if (a > b) is true, b++ does not be executed

Assignment Statements

  • General Syntax: <target_variable> <assignment_operator> <expression>

  • Assignment Operators

    • Most of languages: =
    • Pascal and Ada: :=

  • Conditional Targets

  • Compound Assignment Operators

    • a shorthand method of specifying a commonly needed form of assignment
      • a = a + b can be simply written as a += b

  • Multiple Assignments

    • Python, Perl, and Ruby allow multiple-target multiple-source assignments
      • a, b, c = 1, 2, 3: assign 1, 2, 3 to a, b, c
      • a, b = b, a: swap a, b

  • Mixed-Mode Assignment

    • allows values of different types to be assigned to each other
