Functional Programming

Created2023-04-29|Updated2023-04-29|CSCI3180
|Post View:

Functional Programming

  • supports another form of declarative programming

    • the building block is true function (no side effect)

  • The conceptual model of functional programming is that of a pocket calculator

    • enter an expression, the calculator returns its value

Properties

  1. No Variables Allowed
    • there is no variable like those we have in imperative programming
    • we can only declare names and assign values (like constant)
    • we can reuse the same name but cannot mutate the value

  1. Functional Value

    • the last expression (nameless function)

    • name the function

  1. Typing in ML

    • ML is rather picky about types

    • ML is a highly strongly-typed language in that it requires types of operators and operands to be consistent

      • multiplication is either between integers or between reals, but not between an integer and a real

    • Solution:

      1. we can transform the integer to its real couterpart
      2. we can also apply a built-in function of type int -> real

  1. Basic Types

    1. int

      • 0, 1337, ~3
      • Operations: +, -, *, div, mod

    2. real

      • 3.14, 2E4, ~123.4
      • Operations: +, -, *, /

    3. bool

      • true, false
      • Operations: not, and, or, andalso, orelse
      • Noted: andalso and orelse use short-circuit evaluation

    4. string

      • "doge", "is", "awesome"
      • Operations: ^ (conjunction)
  1. Type Inference
    • ML can infer the types of an expression without the user declaring any time

Conditional Expression

  • Free standing conditional expression

  • Examples:

    • Absolute value abs

    • Check negative negative

Tuples

  • values can be combined into tuples

  • Note: every function in ML is one-argument function

    • it looks like a multi-argument function, but the argument is just a tuple

Selector Functions

  • it is used to explicitly select the components of a tuple
  • underscore _ is the wildcard symbol used in pattern matching (it would ignore the type)

Local Declaration

  • declare a value that have to be local in the function

  • Example: Area of a triangle

    • given that $p$ can simplify a formula
    • since $p$ is only relevant for the computation of $Area$, it is suitable for setting $p$ to be local

  • we can use let for local declaration

  • Local declaration can be used in any expression (not just in function declaration)

User-Defined Types

Enumeration

  • ML supports user-defined types
  • following example is an enumeration type

Case Expression

  • case expression is often used naturally in conjunction with enumeration types

    • using | to seperate the different cases

    • Example:

      • Noted: The order in case expression is very important, because it has the checking order

Union

  • a data type that can represent different data types with same memory storage
  • we can define a union using of and |

Polymorphic

  • we can define polymorphic data types
  • we declare a binary tree using polymorphic data types ('a)

List

  • [] denotes empty list
  • Every non-empty tree contains
    1. The first element (head)
    2. The list of remaining elements (tail)

Combination

  • we can combine the head and tail using the cons operator ::, written as H::T

  • we can combine two lists using @ operator (append)

Homogeneous

  • Note: Lists in ML are homogeneous elements of a list must be same type (type sensitivity)

Operations

  • Build-in function hd() (get the head element) and tl() (get the remaining list)

Pattern Matching

  • using selectors if-then-else to implement summation

  • using pattern matching instead of selectors if-then-else to implement summation

Higher-Order Functions

  • functions that take functions as arguments and/or produce functions as values (because function is actually a value)
    • the function double takes f as input (argument) and return another function that always return twice the result of f

map

  • map function that takes as input a function f and a list [a1, ..., an], and produces the list [f(a1), ..., f(an)] as output

foldl & foldr

  • It takes as input a function f, an initial value x, and a list [a1, ..., an]
  • foldl produces as output the value f(an, f(an-1, ... , f(a2, f(a1, x))))

  • foldr produces as output the value f(a1, f(a2, ... f(an-1, f(an, x)...)))

  • Note that the following implementation is wrong as foldl does not expect take 2 arguments for the function f

  • We can use the functions in other packages
