CSCI3130 - Finite Automata

Created2022-09-11|Updated2022-09-11|CSCI3130
|Post View:

Deterministic Finite Automata

DFA’s Tuples

  1. $ Q $ ~ a finite set of states
  2. $ \Sigma $ ~ an alphabet
  3. $ \delta : Q \times \Sigma \rightarrow Q $ ~ the transition function
  4. $ q_0 \in Q $ ~ the initial state
  5. $ F \subset Q $ ~ the set of accepting states (the final states)
  • Remind: In the diagrams, the accepting states are denoted by double circles.

Language of a DFA

  • A DFA accepts a string if starting from the initial state, following the transition, and ending at the accepting state.
  • The language of a DFA is the set of all string that accepted by that DFA.

Example

$ \Sigma = \left\lbrace 0, 1 \right\rbrace $ , $ Q = \left\lbrace q_0, q_1, q_2 \right\rbrace $ , $ F = \left\lbrace q_0, q_1 \right\rbrace $

transition function $ \delta $:

  • 0 and 011 are in the language.
  • 10 and 0101 are not.

Nondeterministic Finite Automata

NFA’s Tuples

  1. $ Q $ ~ a finite set of states
  2. $ \Sigma $ ~ an alphabet
  3. $ \delta : Q \times (\Sigma \cup \left\lbrace \varepsilon \right\rbrace ) \rightarrow $ subsets of $ Q $ ~ the transition function
  4. $ q_0 \in Q $ ~ the initial state
  5. $ F \subset Q $ ~ the set of accepting states (the final states)
  • Differences from DFA:
    1. transition function can go into several states
      • ie. each state can have zero, one, or more outgoing transition labeled by the same symbol.
    2. allows ε-transition
      • ie. ε-transition can be taken freely (ε = empty string).

NFA Language

  • A NFA accepts a string if there is some path that, starting from the initial state, and ending at the accepting state.
  • The language of a NFA is the set of all string that accepted by that NFA.

Example

$ \Sigma = \left\lbrace 0, 1 \right\rbrace $ , $ Q = \left\lbrace q_0, q_1, q_2 \right\rbrace $ , $ F = \left\lbrace q_2 \right\rbrace $

transition function $ \delta $:

  • ε, 00, 001 and 101 are in the language.
  • 11, 011 is not.
