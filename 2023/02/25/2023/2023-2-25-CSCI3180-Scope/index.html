CSCI3180 - Scope

Created2023-02-25|Updated2023-02-25|CSCI3180
|Post View:

Scope

  • variables and functions declared in a particular scope are visible and accessible only within that scope and its nested scopes, but not outside of it

    1. Local variables

      • The variables defined within a particular function or block of code
      • The are only accessible within that function or block and its nested scopes

    2. Nonlocal variables

      • The variables defined in a parent function
      • They are accessible within nested functions

Static Scope

  • It done before run time

  • It searches locally, then proceeds to increasingly larger enclosing scopes until the given variable name is found

    • Static ancestors: the enclosing static scope that contain it

    • Static parent: the nearest static ancestors

  • Evaluation:

    • Advantages:
      1. Easy to read and understand
      2. Relatively fast
      3. Static type checking is possible
    • Disadvantages:
      1. Too much access is possible
      2. Inconvenience
        • As the program evolves, the initial structure is destoryed and local variables often become global, even subprograms also toward become global rather than nested

Blocks

  • A method of creating static scopes inside a program unit (C++, Java cannot define a function inside a function)

  • Compound statement is a statement sequence that groups more statements into a single statement (in a brace{})

    • each compound statement have declarations which defined a new scope

  • Nested block

    • Reusing name in nested blocks is only legal in C and C++

Declaration Order

  • Some old languages (C89) require variable declaration at the top of the function

    • In C89 compiler, the variable should be declare before the loop

  • C99, C++, Java, C# allow variable declarations appear anywhere

    • In C99, C++, Java, the scope of all local variables is from the declaration to the end of the block

    • But in C#, the scope of any variable declared in a block is the whole block

Global Scope

  • The scope in which variables and functions are visible to the entire program (including in other files)

Python Scope

  • Global variable

    • It can be referenced in all functions
    • It can be assigned if it has been declared to be global using the keyword global in the function

  • Nonlocal variable

    • It can be referenced in the inner functions
    • It can be assigned if it has been declared to be nonlocal using the keyword nonlocal in the function
    • NOTED: The nonlocal only refers to the nearest static ancestors
    • NOTED: The nonlocal only works for nonlocal variables, which means it cannot refer to a global variable

Dynamic Scope

  • The scope of a variable is determined by the order of function calls at runtime rather than by the static structure of the program

    • Temporal
    • Can only be determined at runtime

  • Evaluation:

    • Advantages:
      1. Convenience
        • Variables from the caller are implicitly visible in the callee
    • Disadvantages:
      1. Reliability
        • The variables are visibile to all subprograms it calls
        • Impossible to statically type check
      2. Hard to understand
        • the variable value can be different every time
      3. Costly runtime

Referencing Environments

  • A data structure that stores all variable names that are visible at the statement

  • Static-scoped language:

    • local variables + all visible variables in all enclosing scopes

  • Dynamic-scoped language

    • local variables + all visible variables in all active subprograms in calling sequence
    • active subprograms<: the execution has begun but not yet terminated

  • Practice 1:

    Point Referencing Environment
    (1) local h (reference & assignment), global g (reference only)
    (2) local c (reference & assignment), global g (reference and assignment), nonlocal h (reference only)
    (3) local g (reference & assignment), nonlocal c from sub2 (reference and assignment)

  • Practice 2:

    Point Referencing Environment
    (1) a and b (sub1), c (sub2), d (main)
    (2) b and c (sub2), d (main)
    (3) c and d (main)

Named Constants

  • A variable that is bound to a value only once

  • The binding of values can be either static or dynamic

  • Advantages

    1. Readibility
    2. Reliability
    3. Possibility to parameterize a program

  • Syntax:

    • Java

      • final: dynamically bound

    • C++

      1. const: dynamically bound
      2. constexp: statically bound, must be evaluated at compile time

    • C#

      1. readonly: dynamically bound
      2. const: statically bound, must be evaluated at compile time

Scope & Lifetime

  • Scope and lifetime are often closely related, but are different concepts
  • Example:
    • Lifetime: x is static storage because of the keyword static, because all static storage variables are already existed in the data region
    • Scope: x cannot be accessed in the main function because of its scope, which is not related to the static storage
