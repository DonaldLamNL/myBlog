Image Classification

Created2023-06-26|Updated2023-06-26|CS231N
|Post View:

Image Processing

Challenges

  1. Semantic Gap
    It is a problem for image classification because an image is just a big grid of numbers between 0 and 255
    (e.g., 800 x 600 x 3 channels RGB), which is very difficult to distill the cat-ness from a batch of numbers. (There is a huge gap between the semantic idea of an object and the pixel values in computer)

  1. Viewpoint variation
    When the viewpoint moves, every single pixel values are completely different, but it still represents the same cat.
  1. Complex Image Classification
    • Illumination: There could be different lighting conditions in the scene.
    • Deformation: The object may has different varied poses and positions.
    • Occlusion: Only a part of the object appear in the images.
    • Background Clutter: The obejct may be quite similar to the background.
    • Intraclass Variation: The notion of an object may span a lot of different visual appearances.
    • Context: Objects under different situations may have different visual appearances.

Data-Driven Approach

  1. Collect a dataset of images and labels
  2. Use Machine Learning algorithms to train a classifier
  3. Evaluate the classifier on new images

Nearest Neighbor

It works by finding the most similar data points or instances in a training dataset to a given input or query point. The distance metric includes L1 distance.

  • K-Nearest Neighbor
    Instead of copying label from nearest neighbor, take majority vote from K closest points. The distance metric includes L1 and L2 distance. K Nearest Neighbor Classifier Demo
  • Hyperparameters
    Hyperparameters are the choices or settings that are not learned from the data during the training process but are set by the user before running the algorithm. In Nearest Neighbor classifier, the choices of K and distance metric are the hyperparameters.
    The choose of hyperparameters is a very problem-dependent, it is necessary to try them all out and see the best performance.
    • Idea #1: Choose hyperparameters that work best on the data (not suggested)
      In machine learning, we don’t care about fitting the training data, but how the classifier will perform on unseen data after training.
    • Idea #2: Split data into train and test; choose the hyperparameters that work best on test data (also not suggested)
      It has no idea how the algorithm will perform on new data, which performance is not representative of the performance on the new unseen data.
    • Idea #3: Split data into train, validation and test; choose the hyperparameters that work best on validation and evaluate on test (better)
    • Idea #4: Cross-Validation Split data into folds, try each fold as validation and average the results (useful for small datasets but not frequently used in deep learning)

However, K-Nearest Neighbor is never used on image classification

  1. It is slow at test time
  2. The distance metrics on pixels are not informative
  3. Curse of dimensionality
    The number of training examples that we need to densely cover the space grows exponentially with the dimension.
